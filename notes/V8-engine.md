Deep Dive into V8 JS Engine

## V8 Execution Flow

```text
JavaScript Code
      ↓
Lexical Analysis
      ↓
Tokenization
      ↓
Syntax Analysis
      ↓
AST (Abstract Syntax Tree)
      ↓
Ignition (Interpreter)
      ↓
Bytecode
      ↓
TurboFan (JIT Compiler)
      ↓
Optimized Machine Code
```

---

## Parsing Stage

### Lexical Analysis

* First stage of code execution.
* Breaks JavaScript code into smaller units called **tokens**.

Example:

```js
var a = 10;
```

Tokens:

```text
var → Keyword
a   → Identifier
=   → Operator
10  → Literal
;   → Punctuation
```

### Tokenization

* Process of converting code into tokens.
* Helps V8 understand and process code efficiently.

---

## Syntax Analysis

* Tokens are checked against JavaScript grammar rules.
* Creates an **Abstract Syntax Tree (AST)**.

### AST (Abstract Syntax Tree)

* Tree-like representation of JavaScript code.
* Each node represents a language construct:

  * Variables
  * Expressions
  * Statements

Useful Tool:

```text
https://astexplorer.net
```

### Syntax Errors

* Occur when AST cannot be generated.
* Caused by invalid JavaScript syntax.

---

## Interpreted vs Compiled Languages

### Interpreted Languages

Example:

* Python

Characteristics:

* Executes line by line.
* Faster startup.
* Easier debugging.
* Slower execution.

### Compiled Languages

Examples:

* C
* C++

Characteristics:

* Compiled into machine code first.
* Faster execution.
* Longer compilation time.

---

## Is JavaScript Interpreted or Compiled?

JavaScript is **both**.

### Interpreter

* Executes code quickly.
* Provides fast startup.

### Compiler

* Uses **JIT (Just-In-Time) Compilation**.
* Converts code into machine code during runtime.
* Improves performance.

---

## Ignition (Interpreter)

* V8's interpreter.
* Receives AST.
* Converts AST into Bytecode.
* Executes Bytecode.

Flow:

```text
AST
 ↓
Ignition
 ↓
Bytecode
```

---

## Bytecode

* Intermediate representation of JavaScript code.
* Faster to execute than raw source code.
* Generated by Ignition.

---

## TurboFan (JIT Compiler)

* V8's optimizing compiler.
* Optimizes frequently executed code.

Flow:

```text
Bytecode
   ↓
TurboFan
   ↓
Optimized Machine Code
```

---

## Hot Code

* Code that runs frequently.
* Identified by Ignition.
* Sent to TurboFan for optimization.

---

## Optimization & Deoptimization

### Optimization

* TurboFan makes assumptions about data types.
* Generates highly optimized machine code.

### Deoptimization

Occurs when assumptions fail.

Example:

```js
multiply(2, 3);      // numbers
multiply("2", "3");  // strings
```

Result:

```text
Optimized Code
      ↓
Deoptimization
      ↓
Back to Ignition
```

---

## Important Optimizations

### Inline Caching

* Caches property lookups.
* Speeds up object property access.

### Copy Elision

* Avoids unnecessary object copying.
* Improves performance.

---

## Best Practice

Use consistent data types.

✅ Good

```js
calculate(10);
calculate(20);
calculate(30);
```

❌ Avoid

```js
calculate(10);
calculate("20");
calculate(true);
```

Consistent types help TurboFan keep code optimized.
